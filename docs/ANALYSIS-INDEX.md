## Compliance & Risk Mitigation

**Critical:** All new content must follow `content-compliance.md` rigorously.

### Content Review Checklist (Every Piece)
- [ ] No competitor pricing claims
- [ ] All competitor mentions have "last verified" date (not >6 months old)
- [ ] No false security claims ("100% secure" banned)
- [ ] No client names without written approval
- [ ] Compliance claims use "supports", not "certified"
- [ ] Trademarks attributed (Kong™, MuleSoft®, AWS®)
- [ ] Relevant disclaimer included (compliance, comparisons, security)
- [ ] content-reviewer subagent approved

### Risk Levels
- **P0 (Bloquant):** Diffamation, false claims, client privacy breach → remove immediately
- **P1 (Important):** Outdated comparisons, missing disclaimers → correct within 48h
- **P2 (Suggestion):** Tone improvements, attribution refinement → fix in next editorial pass

See **CONTENT-EXPANSION-COMPLIANCE.md** for full matrix.

## FAQ

**Q: What if we don't have time before the Feb 24 demo?**
A: Prioritize Week 1 content only (freelancer hello world + SMB buying guide). Security whitepaper can start after demo.

**Q: Which segment should we focus on first?**
A: Freelancers (lowest effort, fastest conversion). Then SMBs (higher ACV, longer sales cycle). Then security (highest deal size, but takes 2-3 months).

**Q: Do we need a managed SaaS tier?**
A: Not immediately. Self-hosted positioning is strong. SaaS tier in Q2 would unlock SMB segment and increase ARR/customer.

**Q: What if a blog post gets outdated?**
A: Monthly compliance audit (see CONTENT-EXPANSION-COMPLIANCE.md). Update "last verified" dates, correct facts, republish with note in footer.

**Q: How do we prevent content from going stale?**
A: Set up automation:
- Monthly: Check "last verified" dates in competitor claims
- Quarterly: Re-verify threat model against latest OWASP guidelines
- Semi-annually: Full security whitepaper review
